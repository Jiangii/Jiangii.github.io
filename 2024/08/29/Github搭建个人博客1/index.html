前言

Github Pages

GitHub Pages 是由 GitHub 官方提供的一种免费的静态站点托管服务，让我们可以在 GitHub 仓库里托管和发布自己的静态网站页面。

Hexo

有大佬将博客的功能（比如文章管理）抽象出来，做成了站点生成器 ，可以用极少的"命令" + "配置文件"的方式，自动生成精美的博客网站。主流的博客主要分静态博客和动态博客两类，所谓动态，就是指有前端有后端，可以登录到后台进行管理。静态博客是纯前端的展示，没有后端。

  • 动态博客生成器：WordPress、FarBox、Ghost等
  • 静态博客生成器：Hexo、Jekyll、Octopress、Hugo等

Hexo是一个快速、简洁且高效的博客框架，用于创建和管理静态网站和博客，可以在几秒钟内从数百个文件生成静态网页
官网： hexo.io
官方详细部署文档： 文档|hexo

准备GitHub

1.注册Github

首先注册一个自己的Github账户 官网 | Github

如果Github界面打不开，请自行开启梯子

创建Github仓库

注册之后，接下来创建自己的仓库

  • 首先点击[New]

  • 然后创建仓库时，仓库名一定要以[你的用户名].github.io 为名

  • 创建成功后，可以在仓库项目[Settings]查看相关设置，如果名字一开始没有按照格式设置，可以重新修改

  • 访问我们的默认界面（用户名.github.io）

本地构建Hexo

官方详细部署文档： 文档|hexo

安装Node.js

  1. 进入node.js官网，下载exe安装包
  2. 双击安装包，默认点击下一步进行安装（确保勾选Add to PATH选项）
  3. 安装成功后进入终端输入npm --version显示版本号则说明安装成功

安装Git

  1. 进入git官网，下载exe安装包
  2. 双击安装包，默认点击下一步进行安装（确保勾选Add to PATH选项）
  3. 安装成功后进入终端输入git --version显示版本号则说明安装成功

安装Hexo

所有必备的应用程序安装完成后，即可使用 npm 安装 Hexo

进阶安装和使用
对于熟悉 npm 的进阶用户，可以仅局部安装 hexo 包

安装以后，可以使用以下两种方式执行 Hexo：

  1. npx hexo 命令
  2. Linux 用户可以将 Hexo 所在的目录下的 node_modules 添加到环境变量之中即可直接使用 hexo
    echo ‘PATH=“$PATH:./node_modules/.bin”’ >> ~/.profile
    安装成功后输入hexo --version显示版本号则说明安装成功
