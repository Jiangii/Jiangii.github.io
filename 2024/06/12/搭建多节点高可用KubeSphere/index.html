环境准备

Docker安装

您的集群必须有一个可用的容器运行时。如果您使用 KubeKey 搭建集群，KubeKey 会默认安装最新版本的 Docker。或者，您也可以在创建集群前手动安装 Docker 或其他容器运行时。

依赖项准备

当sudo apt-get update报错

搜索了一下，大多数回答是说镜像错了，需要换镜像，但是测试后发现还是错误的，经过调研，发现那些回答都有问题，只是说要换源，没有说明镜像源要和系统版本一致，所以导致换了他们的源后还是错误的：
例如：输入指令lsb_release -a 查看ubuntu的系统版本，在这里我的系统是22.04.3 LTS，注意后面的jammy这一个版本非常重要。


查看阿里的镜像网站https://developer.aliyun.com/mirror/ubuntu ，发现有三个不一样的源，所以failed to fetch的 原因是镜像源和系统版本对不上 ，所以出现下载错误。
编辑相关配置内容vim /etc/apt/sources.list，更换源，一定要选择与Codename对应的源地址

若是还不行，添加Google 提供的公共 DNS 服务器的 IP 地址 8.8.8.8。
直接在 /etc/resolv.conf 文件后面追加命令（修改文件 sudo vi /etc/resolv.conf）

下载KubeKey

创建集群

  • 安装 KubeSphere 3.4 的建议 Kubernetes 版本：v1.20.x、v1.21.x、v1.22.x、v1.23.x、* v1.24.x、* v1.25.x 和 * v1.26.x。带星号的版本可能出现边缘节点部分功能不可用的情况。因此，如需使用边缘节点，推荐安装 v1.23.x。如果不指定 Kubernetes 版本，KubeKey 将默认安装 Kubernetes v1.23.10。
  • 如果您在这一步的命令中不添加标志 –with-kubesphere，则不会部署 KubeSphere，只能使用配置文件中的 addons 字段安装，或者在您后续使用 ./kk create cluster 命令时再次添加这个标志。
  • 如果您添加标志 –with-kubesphere 时不指定 KubeSphere 版本，则会安装最新版本的 KubeSphere。
